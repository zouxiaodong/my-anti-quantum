**版本**: 2.0  
**日期**: 2026-03-30

## 系统架构

```
Android客户端 → Gateway(8080) → Mock Encryptor(8101)
```

## 启动服务

```bash
# 1. 启动Mock加密机
cd quantum-mock-encryptor
mvn spring-boot:run

# 2. 启动Gateway
cd quantum-server  
mvn spring-boot:run
```

## 算法选择说明

### 密钥协商算法 (Kyber - KEM)

| 算法 | 安全级别 | 适用场景 |
|------|----------|----------|
| Kyber512 | AES-128 | 普通商业数据 |
| Kyber768 | AES-192 | 敏感商业数据 |
| Kyber1024 | AES-256 | 高度敏感数据 |

### 加密算法 (SM4)

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| SM4-CBC | 需要IV，安全性高 | 通用数据加密 |
| SM4-ECB | 简单，相同明文→相同密文 | 单块数据 |

### 签名算法 (Dilithium - 数字签名)

| 算法 | 安全级别 | 适用场景 |
|------|----------|----------|
| Dilithium2 | AES-128 | 普通签名 |
| Dilithium3 | AES-192 | 敏感数据签名 |
| Dilithium5 | AES-256 | 高安全签名 |

## API接口对照表

### 会话管理接口

| 功能 | API调用 | 说明 |
|------|---------|------|
| 新建会话 | 本地状态管理 | 重置所有密钥状态 |
| 获取会话状态 | 本地状态 | 返回当前会话阶段 |

### 密钥协商接口

| 功能 | API调用 | 算法支持 |
|------|---------|----------|
| 生成PQC密钥对 | POST /api/crypto/pqc/genKeyPair | kyber512/kyber768/kyber1024 |

### 随机数接口

| 功能 | API调用 | 参数 |
|------|---------|------|
| 生成随机数 | POST /api/crypto/genRandom | length: 32 |

### 加解密接口

| 功能 | API调用 | 参数 |
|------|---------|------|
| SM4加密 | POST /api/crypto/sm4/encrypt | algorithm: SM4/CBC/NoPadding 或 SM4/ECB/NoPadding |
| SM4解密 | POST /api/crypto/sm4/decrypt | 同上 |

### 签名验签接口

| 功能 | API调用 | 算法支持 |
|------|---------|----------|
| 生成签名密钥对 | POST /api/crypto/pqc/genKeyPair | dilithium2/dilithium3/dilithium5 |
| 签名 | POST /api/crypto/hmac | 使用私钥签名 |
| 验签 | POST /api/crypto/hmac | 使用公钥验签 |

## 注意事项

1. **Mock加密机默认端口**: 8101
2. **Gateway默认端口**: 8080
3. **Android客户端需要连接Gateway(8080)**，不是直接连Mock(8101)
4. **每次新建会话都需要重新生成密钥对**，确保安全性
5. **会话密钥每次随机生成**，不重复使用
6. **完整流程一键测试**会依次执行：密钥协商→随机数→加密→签名
